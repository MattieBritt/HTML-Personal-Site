Section 4 : Intro to CSS

Changing the Background Color of your Website/Body 

 There are multiple ways to change the background color of your Website/Body element
      You can do this by going to your body element and typing "background-color : __"
      There are many colors to choose from, and you can either list a color by name, or specifically use a hex code

Inline CSS 

 Inline CSS is what it sounds like--it's CSS (styling) written within the actual lines of code, not as a separate document, or even a separate space on the page

Internal CSS 

 Similar to Inline CSS--Internal CSS is CSS that's written on a dedicated space in your HTML document using classes or general selectors (it will most likely go within the "head" part of your doc

External CSS 

 External CSS, my personal favorite, is a separate .css file, specifically for the css styling across your ENTIRE website
      This is useful for separating CSS from HTML, along with having more style options available

CSS Hierarchy 

 CSS, evidently, has a hierarchical system when determining what style elements to use first and last
      First, it will use any inline code present within your HTML file
      Next, it will use any internal CSS code present within your HTML file
      And, finally, it will resort to the actual external CSS file with your styling elements
      Keep this in mind when working with future CSS elements! 

      Important Note : It's best practice to keep your CSS selectors/lines in alphabetical order 
      The reason behind this is to make it easier to find a specific line of code when you might be looking for it later on

CSS Classes 

 Classes are selectors that allow you to call specific elements in order to apply style to them
      For example, if you have multiple images on your website, and you don't want all of them to have a background of, purple, for examplewebsite
      Instead of dealing with it, you can apply classes specifically to the images that you don't want to have a background color of purple

      Imporant Note - Differences between ID's and Classes : In short, classes are used for multiple elements, while an ID can only be used for one element
      Additionally, you can only have one id per element, but you can have multiple classes per element

Section 5 : Intermediate CSS

Favicons 

 Favicons are small images that will appear as the icon of the windowed tab 
      These icons can be anything, custom, or not

Divs 

 A "div" stands for a "Content Division Element", it allows you to divide/split up your content/elements into separate boxes 
      This is helpful for making things look nice with precision spacing, while also being able to effect each box separately

The Box Model 

 The Box Model in HTML/CSS is a model for manipulating Divs, along with other particular elements (namely : Padding, Margin, Border, Width, and Height
      Padding - Padding causes the text to be separated from the border by a specified amount (it effects the area between the border and the text)
      Margin - Margin causes the box/div to be pushed away from any other element by a set amount (it spaces div/boxes from each other)
      Border - Border effects the border (duh) in how it looks and operates, you can make the border thicker, thinner, and on any side 
      Width + Height - Self explanatory, it effects the width and height of the element

Display in CSS and HTML 

 Display is an attribute applied to many elements within CSS, as it sounds, diplay, affects how elements are displayed
      Here are the several types of display :
      Block - An element takes up an enitre line--no other elements can be on the same line as it, however you can change the width of the element
      Inline - An element that only takes up as much space as it needs, and other elements can be on the same line as it, however, you CAN'T change the width of the element
      Block-Inline - An element which takes the best of both worlds (from inline and block), you can both have several elements on the same line, AND edit their width
      None - None removes an element entirely, like it doesn't exist lol "Gone, reduced to atoms--I used the stones to destroy the stones"

Positioning 

 There are several types of positioning in CSS, including :
      Static - The default value, does nothing essentially 
      Relative - The element will move away (in other words, add a margin) towards the top, bottom, left, or right, in relativity to where the element should have been.
      (This is useful if you want to keep the flow of your document the same, as it doesn't move other elements around. However, keep in mind, you are moving it in the opposite direction of what you specify)
      Absolute - The element will move  from  the top, bottom, left, or right, in relativity to the parent document (In essence, you're moving it in the direction of where you're specifying)
      Fixed - The element will follow the user and stay fixed in its position across the website (very good for nav-bars or side-bars that you want to be seen at all times)

      Important Note, Natural Stacking Order - The natural stacking order in CSS dictates that whichever element comes first will be the furthest back (on the z plane)
      Additionally, "children sit on their parents", which can also be interpreted as nesting (elements within elements, divs within divs), dictates that the child element of a parent element will be on top

Tips Moving Forward

  • Using the "width" feature instead of using margins to corral elements on the page
  • When you use the width feature and it messes up the centering of elements, instead of using padding, use "margin : auto"
  • Use the "float" propxerty to wrap text around an image (to the left or right of it)
  • A border will not display unless you specify a border type
  • Padding spaces out an element within its div, while a margin spaces out elements outside of the div
  • "Block-Inline" is good for having multiple elements (or divs) on the same line, with the ability to still edit their width

Z-Index and Stacking Order 

 The Z-Index is the value that each element has which determines where they sit on the visual heirarchy, for example, all elements have a Z-Index of 0 as a base 
      If I changed a particular element to have a Z-Index of -1, it would then go behind all of the elements that had a higher Z-Index
      In lamens terms, the Z-Index value determines whether or not an element will be on top of, or behind, another element (in order)

      Important Note : Using the Z-Index can be complicated because the CSS stacking order plays a role as well, however, if you want to use the Z-Index on an element, it HAS to have a "position : " to work
      In my short amount of experience using it, I've found that the relative position is generally the best to use, but it depends on the circumstances of what you're trying to accomplish

Media Query Breakpoints 

 A media query breakpoint is a native CSS feature that allows you to make changes when certain parameters are met
      For example (and most likely a realistic one), I want to make my text larger when the screen size (resolution) is detected going from a laptop resolution, to a tablet/phone resolution
      It's essentially a boolean and an if/else statement combined

      Example : < @media (max-width : 900px) >

      You can also combined multiple media queries with "and" between queries

Code Refactoring 

 Within coding, as a basic foundational rule, it's important to keep your code as neat (readable) as possible
      To help with this, you want to keep your code "DRY" (DRY : "Don't Repeat Yourself")
      This would fall into the category of keeping the broader additions to your code for the classes and the sections, while keeping the specific additions to individual elements

      A few ways to improve your code through refactoring goes as follows : 
      1. Readability - If an axe murderer was reading your code, how much would they want to murder you after they were finished? (Keep code clean to avoid axe murderers, and angry co-workers)
      2. Modularity - How modular is your code? (How easy is it for you to locate sections, classes, ids, and elements within your website? How easily can you edit your code without effecting the rest of the website?)
      3. Efficiency - How efficient is your code? How fast does your code execute and process?
      4. Length - How long is your code? Self explanatory, doesn't really matter for anything except code golf ;p

      You can also use combined selectors with sections to easily refactor your code over a broad scale, increasing both readabiliy and modularity

Combining Selectors 

 Combining CSS selectors allows you to get more specific within your code, here are some ways you can combine CSS selectors :
      Multiple Selectors - A good example of this is in the Tindog project (Tinder Clone)--located within the stylesheet there is a selector targeting h1 through h6, using commas
      Ex : selector1, selector2 {}

      Hierarchical Selectors (Read right to left, child --> parent) - The first selector is from the parent element and the second selector is from the child element, this type of selection allows you to get very specific with your code
      Ex : selector1 selector2 {}

      Combined Selectors (Read left to right, occurs within the same element) - An id or class being combined with an html element, for example
      Ex : h1#title {color : red;}

Selector Priority 

 Here is the order of CSS selector priority (in other words, which addition to your code will actually visibly apply)
      HTML Selector --> CSS Class --> CSS ID --> Inline CSS 
    
      Some tips to avoid running into priority errors are :
      1. Avoid using ID's over classes (use ID's sparingly)--just because there's only one of something doesn't mean it should get an ID (sections are good for ID's)
      2. Only use one class when applying classes to your elements (when using multiple classes on the same element (that all contribute something different to the element), it can easily result in errors)
      3. Avoid Inline Styles AT ALL COSTS! (I don't like using them anyways haha)

Tips Moving Forward, P.3

  • To clarify, and re-iterate, don't target specific ids, classes, or elements, unnecessarily--instead, target the highest parent div/element you can
  • When using a link with an included href, you can use ids of specific sections within the page that will bring them to that part of the page
  • Instead of using width in place of padding, just use padding instead, only use width for its intended purpose (to limit the width of an element) (seems like common sense, but, live and learn lol)
